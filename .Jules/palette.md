## 2025-02-15 - ARIA Live Regions for Lobby States
**Learning:** In a waiting lobby where state updates dynamically without page reloads (e.g. number of ready players), screen readers might remain quiet, causing confusion. Using `aria-live="polite"` coupled with `role="status"` or `role="alert"` (for errors) guarantees that state updates are properly announced to the user. Additionally, hiding decorative SVGs with `aria-hidden="true"` and complementing the native `disabled` attribute with `aria-disabled` provides robust context to AT.
**Action:** When working on real-time interactive dashboards or waiting lobbies, proactively insert `aria-live` regions for any text that changes based on multi-user input and remember to explicitly obscure purely visual loading/state icons using `aria-hidden="true"`.
